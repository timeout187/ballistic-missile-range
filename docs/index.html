Build manifest
  • DONE
    6DOF physics core
    Quaternion attitude, rigid-body rotational dynamics, point-mass ECI gravity, 1976 Standard Atmosphere, Mach/α-dependent aero valid across the full 0–180° angle-of-attack range.
  • DONE
    Guidance & control
    Gravity-turn / lofted / depressed pitch programs, torque-saturated attitude controller, multi-stage boost with staging, uncontrolled ballistic re-entry.
  • DONE
    Analysis module
    Apogee, burnout, max-Q, max Mach, max axial load, impact point, static-stability read-out, CSV/JSON export.
  • DONE
    Animated SVG report
    The panel above — flight profile, ground track, and five synced telemetry channels with a scrub/play transport.
  • DONE
    Streamlit GUI
    Preset and per-stage vehicle editor, launch/guidance parameters, 3D WebGL trajectory view, data export — runs locally, see below.
Scope & known limitations
  • Aerodynamic coefficients are a simplified, editable engineering model — not wind-tunnel or CFD-derived data.
  • Guidance is open-loop; there is no closed-loop range-targeting / impact-point-prediction solver.
  • Gravity is point-mass (no J2 oblateness term); Earth is treated as a sphere.
  • Stage inertia is estimated as a uniform slender cylinder from mass, diameter and length.
  • Two historical presets (Scud-B, Al-Husayn) had their sourced thrust corrected upward to a flyable thrust-to-weight ratio — documented in sixdof/presets.py.
Full interactive GUI

This page is a static replay of one saved flight - GitHub Pages can't run a Python backend. The full interactive app - any preset or fully custom vehicle, live, editable stages and launch parameters - is hosted for free:

sixdof-missile-range.streamlit.app →

Or run it on your own machine:

pip install -r requirements.txt
streamlit run app.py

See the Quick Start wiki page for a full walkthrough.
